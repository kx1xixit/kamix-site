Version 1.1.0-early-access

March 1, 2026 Kane
changelog

This update introduces the shellms manipulation service, the vfs rm command, and a /RAM/busy file to manage command concurrency. It also renames kamix to bootl, separates its commands into the new kamixcs service, and fixes a directory traversal bug in vfs list.

Adds:

  • shellms, the shell manipulation service.
  • New file in RAM (/RAM/busy) that enables or disables new commands from running if a command is already in progress.
  • Finalized vfss command: vfs rm [path].

Changes:

  • Renamed kamix to bootl.
  • Split Kamix commands into a separate service from bootl, named kamixcs.
  • Fixed the bug where vfs list [path] would misinterpret files as directories if your working directory is not / (root).

Removes:

(Nothing.)
